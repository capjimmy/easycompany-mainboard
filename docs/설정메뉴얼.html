2. 사전 셋팅 (테스트 진입 전 필수)

아래 순서대로 셋팅해야 의존 메뉴가 정상 동작합니다. 회사 → 부서 → 사용자 → 권한 → 차량/공간 → 문서 순서를 권장합니다.

2-1. 회사 셋팅

관리 → 회사관리 (슈퍼관리자 전용)

필드설명필수
회사명화면 상단 회사 전환 드롭다운에 표시O
사업자번호세금계산서 발행자/수취자 정보에 사용권장
대표자명세금계산서/증명서 머지 필드권장
주소 / 업태 / 종목세금계산서/공식 문서 머지선택
현재 등록 회사: 건설경제연구원, 건설환경연구소, 이지컨설턴트, 이지건축사사무소, 어반브릿지파트너스

2-2. 부서 셋팅

관리 → 부서관리

  1. 회사 선택(슈퍼관리자) 후 부서 등록.
  2. 부서장(department_manager) 1명을 부서마다 지정 — 연차 1차 승인자가 됨.
  3. 한 사용자가 여러 회사/부서에 속할 수 있음 (junction 테이블 user_companies, user_departments).

2-3. 사원 셋팅

관리 → 사용자관리

필수 / 권장 필드

필드설명영향 범위
username 아이디로그인 ID (영문)로그인

연차 자동 계산 규칙

  • 입사 1년 미만: 근무 월수 × 1일 (최대 11일)
  • 입사 1년~2년: 15일
  • 이후 매 2년마다 +1일 (최대 25일 cap)
  • annual_leave_override가 설정된 경우 위 규칙 무시

비밀번호 초기화

  • 회사관리자/슈퍼관리자가 사용자관리 화면에서 "비밀번호 초기화" → 신규 비밀번호 또는 기본값 password123 설정
  • 회사관리자는 자기 회사 사용자만 가능
  • 슈퍼관리자 비밀번호는 슈퍼관리자만 초기화 가능
테스트 계정 만들 때 입사일을 다양하게 설정하면 연차 계산 검증이 쉬움 (예: 6개월 전, 1년 전, 5년 전, 11년 전).

2-4. 권한 셋팅

관리 → 권한설정 / 권한설정 → 매트릭스

2가지 권한 모드

  1. 역할 기본값 (DEFAULT_PERMISSIONS) — 사용자별 권한이 비어있으면 역할 기본값 적용. 위 1-2 매트릭스 참고.
  2. 사용자별 덮어쓰기 — 권한설정 화면에서 사용자를 선택하고 메뉴 키별로 view/create/edit/delete 토글. 한 항목이라도 설정되면 해당 사용자의 권한 전체가 덮어쓰기 모드로 전환.
권한설정 검증
  • 매트릭스 화면에서 행=사용자, 열=메뉴 키 토글 가능
  • 토글 후 새로고침 시 즉시 반영 (해당 사용자 재로그인 또는 새 세션)
  • 덮어쓰기된 사용자에게서 모든 키 OFF 시 메뉴 자체가 사이드바에서 사라짐
  • 역할이 변경되면 기본 권한이 자동으로 새 역할의 DEFAULT_PERMISSIONS로 적용

2-5. 차량 / 공간 셋팅

관리 → 차량관리 / 공간관리

  • 차량: 차량번호 필수, 차종/모델/색상 선택. 슈퍼관리자는 "공통(전 회사 공유)" 옵션으로 등록 가능.
  • 공간: 회의실/강의실. 활성 상태 ON인 공간만 캘린더에서 예약 가능.
  • 둘 다 회사별/공통 등록 지원. 데이터는 운행일지 / 공간 캘린더에서 사용.

2-6-1. 문서 템플릿

시스템 → 문서 템플릿

템플릿 종류용도출력 형식
견적서견적관리에서 다운로드.xlsx / .docx / .hwpx

템플릿 파일 안에 머지 필드 사용: {{회사명}} {{거래처명}} {{계약금액}} {{name}} {{rank}} {{hire_date}} 등. 미리보기 모달에서 변수 목록 확인 가능.

2-6-2. 단가 설정

시스템 → 단가 설정

  • 견적서 작성 시 자동으로 호출되는 항목별 단가 카탈로그
  • 회사별 / 카테고리별로 등록 — 슈퍼/회사관리자가 등록

2-6-3. 백업 (선택)

시스템 → 백업관리

  • 전체 DB 스냅샷을 zip으로 다운로드 (관리자만)
  • 복원 기능은 슈퍼관리자만 노출
테스트 진입 전 최종 체크리스트
  • 회사 1개 이상 등록 / 부서장 지정된 부서 1개 이상
  • 역할별 테스트 계정 1개씩 (super_admin, company_admin, department_manager, employee)
  • 계정마다 hire_date / department_id 셋팅 완료
  • 차량 1대 이상, 공간 1개 이상 (활성 상태)
  • 문서 템플릿 (견적/계약/세금계산서) 최소 1개씩 업로드
  • 단가 카탈로그 항목 1개 이상 (견적서 테스트용)

3-1. 연차 신청 / 승인

3-1-1. 신청자 화면

인사/총무 → 연차 신청

휴가 유형차감비고
연차 (annual)일수만큼 차감기본값
오전반차 / 오후반차0.5일 차감일수 자동으로 0.5 설정
병가 (sick)차감 없음증빙 별도 관리
특별휴가 (special)차감 없음경조사 등

T3.1 신청 → 일수 자동 계산
  • 휴가 유형 = 연차, 기간 = 3일 → 사용일수 자동 3일
  • 유형을 오전반차로 변경 → 자동 0.5일
  • 병가 5일 → 잔여연차 차감 안 됨
T3.2 잔여연차 카운터
  • 총 / 사용 / 잔여 카드가 입사일 기반으로 표시
  • 잔여 ≤ 3일이면 빨간색
  • 승인된 연차 취소 시 잔여 즉시 환원

3-1-2. 승인자 화면 (2단계 승인)

인사/총무 → 연차 승인관리

단계상태 전이처리 권한
1. 신청(없음) → 대기본인
2. 부서승인대기 → 부서승인department_manager
3. 최종승인부서승인 → 최종승인company_admin / super_admin
3'. 직접승인대기 → 최종승인company_admin / super_admin
X. 반려임의 → 반려해당 단계 권한자
T3.3 부서장 1차 승인
  • 부서장 계정으로 로그인 → 본인 부서원의 신청만 보임
  • "부서승인" → 상태 전환 후 회사관리자 화면에 노출
T3.4 회사관리자 최종승인

  • 최종승인 시 신청자 잔여연차 즉시 차감
  • 반려 시 사유 입력 모달, 신청자에게 사유 노출

4. 재무 / 회계

재무/회계 메뉴는 사원에게는 지출결의서를 제외하고 노출되지 않습니다. 부서관리자 이상으로 테스트.

4-1. 미수금 관리

재무/회계 → 미수금관리

계약별로 청구된 금액 중 아직 입금되지 않은 잔액을 한눈에 봄.

T4.1 미수금 조회
  • 거래처 / 부서 / 기간 / 상태 필터 동작
  • 총 청구 / 입금 / 미수 합계 카드 표시
  • 회사 전환 시 데이터가 회사별로 격리
  • 모든 컬럼 드래그로 폭 조절 가능 (ResizableTable)

4-2. 청구 / 입금

재무/회계 → 청구/입금

계약별로 청구 단계(선금 / 중도금 / 잔금)와 입금 내역을 등록.

T4.2 청구 등록
  • 계약 검색/선택 → 청구 단계, 청구일, 청구금액 입력
  • VAT 포함/별도 옵션 동작 (포함=금액÷11, 별도=금액×0.1)
  • 등록 후 미수금관리 카드에 즉시 반영
T4.3 입금 등록
  • 청구 행 → "입금 등록" → 입금일/금액 입력
  • 부분 입금 가능 (미수금이 줄어드는 만큼만)
  • 수금률이 자동 계산 (입금합계 ÷ 총청구액)
  • 100% 초과 시 경고 또는 거부

4-3. 세금계산서

재무/회계 → 세금계산서

매출(발행) / 매입(수취) 양방향 관리. 계약 / 외주와 연결.

방향의미비고

T4.4 발행 등록 + 다운로드
  • 발행/수취 탭 전환 동작
  • 계약 검색 → 자동으로 거래처/사업자번호/공급가액 채움
  • VAT 자동 계산 (10%)
  • 저장 후 .xlsx 다운로드 (템플릿 머지 검증)
  • 모든 컬럼 드래그로 폭 조절 가능

4-4. 미지급금 관리

재무/회계 → 미지급금관리

외주 발생 후 아직 지급하지 않은 금액 추적.

T4.5 미지급 조회 / 지급 처리
  • 외주관리에서 등록한 외주가 미지급금에 자동 노출
  • 지급 등록 → 지급일 / 지급금액 → 미지급 잔액 차감
  • 거래처/기간 필터, 합계 카드 동작

4-5. 경비 정산

재무/회계 → 경비정산

회사 차원의 경비 정산 (지출결의서와는 별개로 회사관리자가 일괄 등록).

T4.6 경비 등록
  • 일자 / 부서 / 항목 / 금액 / 증빙 입력
  • 월별 / 항목별 합계 통계 카드 동작

4-6. 지출결의서

재무/회계 → 지출결의서

사원이 자기 지출에 대해 결의서를 신청하고 회사관리자가 승인하는 플로우.

카테고리 (입력 시 선택)

교통비 / 식비 / 숙박비 / 소모품비 / 통신비 / 회의비 / 접대비 / 교육비 / 택배비 / 제본비 / 기타

승인 플로우

단계상태권한
1. 신청대기사원 / 회사관리자 / 슈퍼관리자
(부서관리자는 신청 불가, 조회만)
2. 승인승인회사관리자 / 슈퍼관리자
X. 반려반려회사관리자 / 슈퍼관리자
T4.7 사원 신청
  • 사원 계정 → "+ 지출결의서 작성"
  • 신청일 / 부서(자기 부서) / 카테고리 / 금액 / 사용 내역 / 증빙
  • 저장 시 대기 상태로 등록 → 회사관리자 화면에 노출
T4.8 관리자 승인
  • 회사관리자 화면 → 부서 필터 + 신청자 이름 검색 동작
  • "승인" / "반려(사유)" 버튼 동작
  • 승인 후 신청자 화면에서 상태 즉시 갱신
  • 지출결의서 .xlsx / .docx 다운로드 (템플릿 머지)

4-7. 가수금 관리

재무/회계 → 가수금관리

입금되었지만 아직 어느 계약/거래처에 대응되는지 미확정인 금액. 매칭 후 청구/입금에 반영.

T4.9 가수금 등록 / 매칭
  • 입금일 / 입금자명 / 금액 / 메모 입력 후 등록
  • "매칭" → 계약/청구 검색 후 적용
  • 매칭 시 가수금 잔액 차감, 청구 입금에 반영

5. 회귀 체크리스트 (역할별 시나리오)

위 단위 테스트가 끝나면 아래 시나리오로 통합 점검.

시나리오 A — 사원
  • 사이드바에 재무/회계 그룹은 지출결의서만 보임
  • 관리 / 권한설정 / 회사관리 메뉴 모두 미노출
  • 견적/계약 신규 작성 가능 (수정/삭제 불가)
  • 연차 1일 신청 → 부서장에게 라우팅
  • 지출결의서 1건 작성 → 대기 상태로 등록
  • 운행일지 1건 등록
시나리오 B — 부서관리자
  • "연차 승인관리"에서 부서원 신청 1차승인
  • 지출결의서 메뉴는 조회만 가능 (작성 버튼 없음)
  • 견적/계약/거래처/외주 편집 가능
  • 미수금/청구·입금/세금계산서/미지급/경비/가수금 모두 편집 가능
  • 관리 / 권한설정 / 회사관리 / 차량관리 / 공간관리 미노출
시나리오 C — 회사관리자
  • 전 부서 연차 → 부서승인된 건 최종승인
  • 지출결의서 승인 + 다운로드
  • 세금계산서 발행 + .xlsx 다운로드
  • 사용자 1명 추가 + 권한 매트릭스에서 1개 메뉴 OFF → 그 사용자 재로그인 시 즉시 반영
  • 차량 1대 + 공간 1개 추가 등록 → 운행일지/공간 캘린더에서 즉시 사용 가능
  • HR 캘린더에서 최종승인 건 표시
시나리오 D — 슈퍼관리자
  • 회사 전환 드롭다운으로 데이터 격리 검증
  • 차량/공간을 "공통"으로 등록 → 모든 회사에서 사용
  • 회사관리에서 신규 회사 등록 / 삭제
  • 슈퍼관리자 본인 비밀번호 초기화 (다른 슈퍼관리자만 가능)
  • 백업 다운로드 → 복원 옵션 노출 확인
  • 매뉴얼 관리에서 본 매뉴얼 등록 / 게시
발견 시 즉시 신고할 결함
  • 잔여 연차가 음수 / 25일 초과로 표시되는 경우
  • 부서장 라우팅이 안 되어 신청이 사라지는 경우 (department_id 누락 의심)
  • 승인된 연차 취소 시 잔여 환원 안 되는 경우
  • 세금계산서 .xlsx 다운로드에 머지 필드({{...}})가 그대로 남아있는 경우
  • 수금률 100% 초과되는 청구/입금
  • 회사 전환 시 다른 회사 데이터가 노출되는 경우
  • 사원 계정에서 재무/회계 메뉴(지출결의서 외)가 노출되는 경우
  • 공간 캘린더에서 시간 충돌 예약이 저장되는 경우
  • 권한 매트릭스 변경이 새로고침해도 반영 안 되는 경우
