4. 데이터 전략: 로컬 → 서버 DATA

로컬 저장
electron-store (JSON)
→
하이브리드
로컬 + API 동기화
→
서버 전환
AWS Lightsail + PostgreSQL

단계별 전환 전략

단계 시점 데이터 저장 위치 설명
Phase 1 현재 ~ 4주차 100% 로컬 electron-store (JSON) 기반 로컬 데이터. 빠른 개발 집중.
Phase 2 5 ~ 7주차 로컬 + 서버 동기화 데이터 서비스 레이어 추가. 로컬에 캐시, 서버에 백업/동기화.
Phase 3 8주차 ~ 서버 중심 PostgreSQL 서버를 메인 DB로 전환. 로컬은 오프라인 캐시만.

모듈화 핵심: 데이터 서비스 추상화

모든 모듈은 DataService 인터페이스를 통해 데이터에 접근합니다. 이렇게 하면 저장소가 로컬→서버로 바뀌어도 모듈 코드를 수정할 필요가 없습니다. 

// 모든 모듈이 사용하는 통일된 인터페이스
interface DataService<T> {
  getAll(): Promise<T[]>
  getById(id: string): Promise<T | null>
  create(data: Omit<T, 'id'>): Promise<T>
  update(id: string, data: Partial<T>): Promise<T>
  delete(id: string): Promise<boolean>
}

// Phase 1: 로컬 구현
class LocalContractService implements DataService<Contract> { ... }

// Phase 2: 하이브리드 (로컬 캐시 + 서버 동기화)
class HybridContractService implements DataService<Contract> { ... }

// Phase 3: 서버 직접 통신
class ServerContractService implements DataService<Contract> { ... }

5. 모듈별 핵심 설계 MODULES

📋 계약관리 (3~4주차)

  • 계약 CRUD: 거래처, 계약명, 금액, 기간, 상태
  • 월별현황: 차트 (매출 추이, 계약 건수)
  • 변경이력: 계약 수정 시 자동 이력 저장
  • 외주관리: 하도급 계약, 외주비 관리
  • → 재무: 계약 등록 시 미수금 자동 생성
  • → 프로젝트: 계약 확정 시 프로젝트 자동 생성

💰 재무/회계 (5~6주차)

  • 미수금관리: 계약별 청구 금액 vs 입금 금액
  • 미지급금관리: 외주비, 경비 미지급 관리
  • 보증금관리: 입찰보증금, 계약보증금 등
  • 청구/입금: 세금계산서 발행, 입금 확인
  • ← 계약: 계약 정보에서 미수금 데이터 수신
  • → 계약: 입금 완료 시 계약 상태 업데이트

👥 인사/총무 (7주차)

  • 직원명부: 기본정보, 부서, 직급, 연락처
  • 연차관리: 연차 부여/사용/잔여 현황
  • 생일관리: 직원/거래처 생일 알림
  • 법인차량: 차량 정보, 보험, 운행일지
  • → 계약/프로젝트: 담당자 정보 제공

📊 프로젝트 (8주차)

  • 프로젝트현황판: 진행률, 일정, 담당자
  • 일정관리: 캘린더 뷰, 마일스톤
  • ← 계약: 계약 확정 시 프로젝트 자동 생성
  • ← 인사: 담당자 배정 정보 수신
  • → 계약: 완료 시 검수 요청 트리거

6. 핵심 설계 원칙 PRINCIPLES

01
모듈 독립성
각 모듈은 독립적으로 개발·배포 가능. 모듈 추가 시 기존 코드 수정 최소화.
02
이벤트 기반 연동
모듈 간 데이터 공유는 이벤트/콜백으로 처리. 직접 참조 없이 느슨한 결합.
03
데이터 추상화
DataService 인터페이스로 저장소 추상화. 로컬→서버 전환 시 모듈 코드 변경 불필요.

모듈 추가 절차 (개발자 가이드)

  1. src/renderer/pages/[모듈명]/ 폴더 생성 → 페이지 컴포넌트 개발
  2. src/shared/constants/menu.ts → MENU_STRUCTURE에 메뉴 항목 추가
  3. src/renderer/App.tsx → Route 추가
  4. src/main/ipc/[모듈명].ipc.ts → IPC 핸들러 추가
  5. src/main/database/index.ts → 데이터 스키마 추가
  6. src/main/index.ts → IPC 핸들러 등록
  7. 대시보드에 해당 모듈 섹션 추가
